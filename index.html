How rates and orders connect behind the scenes (setup notes)
1. Rates from Google Sheet (automatic). Wired to the store's published Sheet — every edit there shows up here the next time the app is opened, no re-upload needed. Expected columns: Code, Name, Marathi Name, Category, SubCategory, Size, MRP, Discount, Image URL. Add a direct image link in Image URL (a shared Google Drive link or any public photo URL) and it shows up on that item's card automatically; leave it blank and the item keeps its colored icon.

2. Orders → Google Sheet. A small Google Apps Script "Web App" bound to the store's order-tracking Sheet accepts each order posted from checkout (its URL goes in ORDER_WEBHOOK_URL) and adds it as a new row, status defaulting to "Packing".

3. "Order ready" WhatsApp message. When staff mark a Sheet row "Packed," an onEdit trigger can call the WhatsApp Business Cloud API with the customer's number — the same kind of setup already running for the RMC/Lanja citizen chatbot.

4. Home delivery rule. Enforced in the cart already — orders above ₹1000 qualify for delivery; the threshold is one constant in the code.

5. Full item catalog. The store's Cloth and Provision & Stationery lists (3,391 items total) were extracted from the PDFs into suk_bazaar_full_item_seed.xlsx — codes, sizes, and ~1,315 prices that were printed directly in the source list. Names need review (best-guess only) and the Grocery list still needs manual entry since its PDF had no readable text. Once reviewed, paste it into the Google Sheet to bring it into the app.
